# Task Description

Document best practices for Git collaboration. Forking vs branching workflows. Creating descriptive pull requests: title, description with issue references, checklist (tests pass, documentation updated, breaking changes noted). Code review etiquette: be constructive, explain why not just what, approve with comments or request changes. Handling review feedback: make requested changes, re-request review. Squash commits before merging for clean history. Show how to link PRs to issues using keywords (Fixes #123).
