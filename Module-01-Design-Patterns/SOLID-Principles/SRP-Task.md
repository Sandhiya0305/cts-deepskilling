# Task Description

Implement the Single Responsibility Principle (SRP). Create a BEFORE example where one class handles multiple responsibilities (e.g., an Employee class that calculates salary, prints reports, AND saves to database). Then create an AFTER example where each responsibility is separated into its own class: SalaryCalculator, ReportPrinter, and EmployeeRepository. Add comments explaining why violating SRP leads to fragile code and how separation improves maintainability.
